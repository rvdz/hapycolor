Lightline¶

A light and configurable statusline/tabline plugin for Vim (see github repository). For a simple configuration, add the following instruction in your vimrc:

let g:lightline = {'colorscheme': 'hapycolor' }

It currently supports various themes inspired from lightline’s repository:

  • Solarized
  • Wombat
  • One
  • Landscape

In order to add new themes, a template should be added in hapycolor/targets/lightline_themes/, written in a flattened [1] format and can use the following undefined variables:

  • s:blue
  • s:magenta
  • s:yellow
  • s:red
  • s:orange
  • s:green
[1]Each color of the theme must be defined as a pair 24bit and 8 bit color: [ ‘#24bit_color’, ‘8bit_color’ ]

Those variables will then be defined when exporting a palette to the target.

How to add a target?¶

In order to add a target, two steps are required:

First, a class extending hapycolor.targets.base.Target needs to be implemented in the module hapycolor.targets. The main method, the static function export(), takes in a palette of colors and exports it to the target. Other methods that should be implemented are:

  • compatible_os(), which defines a list of compatible OS.
  • initialize_config(), which interacts with the user and stores in its respective section of the configuration file persistent data needed to export a palette.
  • reconfigure(). This method can be triggered by hapycolor’s -reconfigure- option, which asks for the name of the target’s module to reconfigure. To do so, it currently searches for a class that matches the module’s name implemented in it, except, named in PascalCase, rather than snake_case in the case of the module.

Finally, the module’s name needs to be added to targets’s environment. In other words, the __all__ list defined in targets.__init__ shoud be altered to include the new target.
